# jiaolong AI 助手框架 — Briefing

> **一句话描述**: 让 AI 助手拥有记忆、会学习、能自我进化的增强框架

---

## 解决的问题

| 痛点 | jiaolong的解决方案 |
|------|-------------|
| AI每次对话都要重新介绍背景 | **语义记忆召回** — 自动理解上下文 |
| 复杂任务要手动拆解执行 | **LLM任务分解** — 自动拆解+并行执行 |
| 代码质量靠人工review | **clean-code规则引擎** — 写代码自动检查 |
| 重复问题要反复回答 | **Skills自动触发** — 关键词即触发 |
| 后台任务需要手动运行 | **Daemon守护** — 定时调度+进程管理 |


---

## 三大核心能力

### 1. 记忆层
- **语义召回**: 不用精确关键词，模糊描述也能找到记忆
- **三层演进**: 热层常用 → 温层归档 → 冷层存档，自动升降级
- **每次对话注入**: 无需手动@recall，AI自动带着上下文

### 2. 自动化层
- **Skills自动触发**: 说"帮我选股"就执行选股，不用记命令
- **并行执行**: 3个任务同时跑，耗时从3秒→1.2秒
- **Daemon守护**: 后台自动运行，不用人工盯着

### 3. 质量层
- **8条clean-code规则**: 空stub/魔法数字/函数过长等自动检测
- **LLM对话压缩**: 长对话→智能摘要，节省50%+ token
- **任务智能分解**: 复杂任务自动拆解为可执行步骤
